# 🔍 コントラクトアドレスの確認方法

## 問題
「コントラクトアドレスが無効です」というエラーが表示される場合、`.env.local`の`NEXT_PUBLIC_TRASH_NFT_ADDRESS`が正しく設定されていません。

## 解決方法

### 1. コントラクトアドレスの形式を確認

**正しい形式（40文字の16進数）:**
```
NEXT_PUBLIC_TRASH_NFT_ADDRESS=0x1234567890123456789012345678901234567890
```

**間違った形式の例:**
- トランザクションハッシュ（64文字）: `0x6ed642fdd4afa60a94a63bbd2ab5aaa4c56bb21afd4966f82bceaf11e1bfbcc8` ❌
- 空のアドレス: `0x0000000000000000000000000000000000000000` ❌
- 短すぎる: `0x1234` ❌

### 2. コントラクトを再デプロイしてアドレスを取得

もしコントラクトアドレスが分からない場合、再デプロイしてください：

```cmd
set PRIVATE_KEY=your_private_key_here
npx hardhat run scripts/deploy-hardhat.js --network baseSepolia
```

デプロイが成功すると、以下のような出力が表示されます：

```
✅ デプロイ完了！
📝 コントラクトアドレス: 0x1234567890123456789012345678901234567890
```

### 3. `.env.local`ファイルを更新

プロジェクトルートに`.env.local`ファイルを作成（または更新）して、以下を設定：

```env
NEXT_PUBLIC_USE_TESTNET=true
NEXT_PUBLIC_TRASH_NFT_ADDRESS=0xデプロイされたコントラクトアドレス
```

**重要:**
- `0x`で始まる42文字（0x + 40文字）のアドレスである必要があります
- トランザクションハッシュではなく、コントラクトアドレスを使用してください

### 4. アプリを再起動

`.env.local`を更新したら、開発サーバーを再起動：

```cmd
npm run dev
```

### 5. 確認

ブラウザのコンソール（F12）で以下を確認：

```
TrashNFT Contract Address (validated): 0x...
Is contract address valid: true
```

`Is contract address valid: false` または `null` が表示される場合は、アドレスが正しく設定されていません。

## トラブルシューティング

### アドレスが64文字の場合

トランザクションハッシュを誤って設定している可能性があります。デプロイ時の出力で「コントラクトアドレス」と表示されているものを使用してください。

### アドレスが表示されない場合

デプロイスクリプトを再実行して、コントラクトアドレスを確認してください。
